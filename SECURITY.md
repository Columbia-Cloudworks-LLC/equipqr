# Security Policy

## 🔒 Supported Versions

We currently support security updates for the following versions:

| Version    | Supported          |
|------------|--------------------|
| `main`     | ✅ Always Supported |
| All others | ❌ Not actively maintained |

We recommend all users stay on the latest version deployed to `main`.

---

## 📢 Reporting a Vulnerability

If you believe you've found a **security vulnerability** in this repository:

1. **Do not open a public issue.**
2. Instead, please email **security@columbiacloudworks.com** with the following:
   - Description of the vulnerability
   - Steps to reproduce
   - Impact assessment (what data or users are affected)
   - Any relevant logs, screenshots, or proof-of-concept code

If you're unable to use email, you may also [create a private GitHub security advisory](https://github.com/Columbia-Cloudworks-LLC/equipqr/security/advisories).

---

## 🔁 What to Expect

- We will acknowledge receipt within **2 business days**
- For valid reports, we aim to investigate and respond within **5 business days**
- If a fix is required, we’ll provide a patch and coordinated disclosure timeline
- Credit will be given to responsible disclosures unless anonymity is requested

---

## 🧪 Safe Harbor

Good-faith researchers will not be penalized for reporting vulnerabilities. Please avoid data destruction, privacy violations, or denial-of-service while testing.

---

Thank you for helping us keep our users safe!
